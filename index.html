基于 Hexo 框架在 Github 上搭建博客

Veröffentlicht am 2018-01-19

壹 序言

Hexo，一个快速、简洁且高效的博客框架，若以个人博文为主导需求，它的强大毋庸置疑。

Hexo 支持 GitHub Flavored Markdown 的所有功能，且基于 Node.Js 所带来的超快生成速度，开放性 API 以及丰富的插件等，即功能来得纯粹，不失个性化定制，上手容易且操作便捷。更重要它是开源的、免费的博客框架，数据保存在本地以及 Github 上，相信在众多博客产品中选择、博弈，不妨首要考虑 Hexo。

若想了解详情可访问「hexo.io」。

贰 搭建环境

  • Mac OS、Windows 环境下的配置流程和步骤大同小异，而搭建环境的大致流程为：
    • Git 环境配置 ( Windows )
    • Node.Js 环境配置
    • Github 账号注册和配置
    • Hexo 的安装和配置
  • 考虑到是流程操作类的文章，为简单起见，则我们以 Windows 和 MacOS 版本分别说明情况：
    • Windows 版本
    • MacOS 版本

MacOS 版本

配置环境

  • Xcode: Hexo的编译依赖于Xcode。

  • Node.js: Hexo是基于Node.js开发的。

    • 官方下载
    • 本地安装，文件包为「Node-Vx.x.x.pkg」

    • 检测是否安装成功，当然返回版本信息即成功安装

  • Github Account ( 已有账号、项目可忽略 )

    • 注册账号「GitHub」

    • 创建代码库

      具体操作步骤:

      1. Add(右上角) > New Repository；
      2. Repository > Name，填写「yourname.github.io」；
      3. Repository > Description，填写「简单描述」；

    • 配置代码库

      具体操作步骤:

      1. 进入项目「yourname.github.io」；
      2. Settings > GitHub Pages，开启「GH-Pages」功能；
      3. 点击「Launch Automatic Page Generator」；

      第三步说明:
      Github 将会自动替你创建出一个 GH-Pages 的页面。若配置没有问题，约15分钟后，yourname.github.io 就可以正常访问了，配置结束。

安装 HEXO

  • 安装 HEXO

    • 本地选择安装目录

  • 在线安装 HEXO ( NPM淘宝源 )

  • 检测是否安装成功，当然返回版本信息即成功安装

  • 运作 HEXO ( 每次发布文章的流程操作 )

    • 本地运作 HEXO

      hexo server // 或简写: hexo s

    • 当终端提示以下信息，则本地配置工作基本完成。

  • 使用 HEXO

    • 新建一篇博文 ( 当然，更新同一篇文章只需修改「Hexo > Source > _Posts」目录下相应的MarkDown文件 )

      hexo new post “Article Title”

    • 生成博文 ( 生成静态网页 )

      hexo generate

    • 部署至 Git 的准备工作

      • 在部署前，还需要安装「Git部署插件」:
        npm install hexo-deployer-git –save

    • 再者，与 Github 连接前需要获得授权，不然会出现以下情况 ( 本机没有配置 Public Key ):

      [注意] 若之前在机器上配置了 Git 的信息，请跳过该步骤。

      • 配置「SSH KEY / Public KEY」

        • 配置 Git 的 UserName 和 Email

          cd ~/.ssh

          列出该目录下的文件，看是否存在「id_isa」和「id_isa.pub」文件

          ls

          没有则继续下一步骤「生成密钥」

          邮箱「UserName@xxx.com」替换成自己的邮箱

          ssh-keygen -t rsa -C “UserName@xxx.com”

          1. Avatar(头像) > Settings > Personal Settings > SSH and GPG keys

          2. New SSH KEY，把「id_isa.pub」的密钥粘贴过去即可

          3. ssh -T git@github.com

          4. 提示「Hi YourName! You’ve successfully authenticated, but GitHub does not provide shell access.」即成功配置。

  • 启用主题

    • 与所有 Hexo 主题启用的模式一样。 当 克隆/下载 完成后，打开 站点配置文件， 找到 theme 字段，并将其值更改为 next。

  • 到此，NexT 主题安装完成。下一步我们将验证主题是否正确启用。在切换主题之后、验证之前， 我们最好使用 hexo clean 来清除 Hexo 的缓存。

    hexo clean
